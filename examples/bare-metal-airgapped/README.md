# Bare Metal Airgapped OpenShift Deployment

This example performs a complete end-to-end airgapped OpenShift 4.x deployment on
bare metal using IPI (Installer-Provisioned Infrastructure).

## Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| `openshift-install` | 4.14+ | Must be present on the Terraform executor |
| `oc` | 4.14+ | Required for `oc mirror` |
| `mirror-registry` | 1.3+ | Quay-based mirror registry installer |
| Terraform | 1.5+ | |

All tools must be accessible on the machine running Terraform, or at the absolute
paths configured in the provider block.


### 2. Initialize and deploy

```bash
# Build and install the provider
make install

# Initialize Terraform
terraform init

# Preview the deployment plan
terraform plan

# Deploy (this will take 90–120 minutes for the cluster)
terraform apply
```

### 3. Access the cluster

```bash
# Export kubeconfig
export KUBECONFIG=$(terraform output -raw kubeconfig_path)

# Check cluster status
oc get nodes
oc get clusteroperators

# Open the web console
echo "Console: $(terraform output -raw console_url)"
echo "Username: kubeadmin"
echo "Password: $(cat $(terraform output -raw kubeadmin_password_path))"
```

## Deployment order

Terraform manages the following dependency chain automatically:

1. **`openshift_mirror_registry`** — installs the internal Quay registry
2. **`openshift_image_mirror`** — mirrors OCP + operator images into the registry
3. **`openshift_install_config`** — generates `install-config.yaml` with mirror settings
4. **`openshift_cluster`** — runs `openshift-install create cluster`
5. **`openshift_catalog_source`** — adds the mirrored operator catalog
6. **`openshift_subscription`** — subscribes to Local Storage Operator
7. **`openshift_machine_config`** — applies kernel arguments for hugepages on workers
8. **`openshift_machine_set`** — provisions dedicated infra nodes

## Airgapped considerations

- The `additional_trust_bundle` in the install-config is automatically populated from
  the self-signed CA cert generated by `mirror-registry install`.
- `imageContentSources` are automatically wired from `oc mirror` output.
- The pull secret must contain credentials for the internal registry **in addition to**
  the Red Hat registries (used during the initial mirroring step from an internet-connected
  machine, if applicable).
- DNS must resolve `api.<cluster_name>.<base_domain>` and
  `*.apps.<cluster_name>.<base_domain>` to the respective VIPs.

## Destroy

```bash
terraform destroy
```

This runs `openshift-install destroy cluster` before removing all other resources.
